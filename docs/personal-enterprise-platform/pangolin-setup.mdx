This document provides a comprehensive guide to installing and configuring Pangolin as an infrastructure management platform on Proxmox VE.

## What is Pangolin?

[Pangolin](https://digpangolin.com/) is a self-hosted infrastructure management platform that provides:
- Server and service monitoring
- Docker container management
- Automated deployments
- Centralized dashboard for infrastructure overview
- Real-time alerts and notifications

## Prerequisites

Before proceeding with the Pangolin installation, ensure you have:

- Proxmox VE host with configured network bridges (see [Proxmox Setup](./proxmox-setup.mdx))
- OPNsense firewall configured (see [OPNsense Setup](./opnsense-setup.mdx))
- Domain name (optional, but recommended for HTTPS access via reverse proxy). This should be your base domain as established in the [Prerequisites for the Personal Enterprise Platform](../prerequisites.md).
- Basic understanding of Docker and containerization

## 1. LXC Container Creation

### 1.1 Container Specifications

Create a new LXC container in Proxmox with the following specifications:

**Basic Settings:**
- **Template**: Debian 13 (trixie) or latest stable (e.g., `debian-13-standard_13.1-1_amd64.tar.zst`)
- **Hostname**: `pangolin`
- **Unprivileged container**: No (Docker requires privileged)
- **Nesting**: Yes (enable in Options after creation)
- **Keyctl**: Yes (enable in Options after creation)
- **Password**: Enter and confirm a strong password. (Alternative: Use a SSH Key)

**Resources:**
- **Recommended**:
  - CPU: 4 cores
  - RAM: 4096 MB (4 GB)
  - Swap: 512 MB
  - Disk: 32 GB
- **Minimum**:
  - CPU: 2 cores
  - RAM: 2048 MB (2 GB)
  - Swap: 512 MB
  - Disk: 16 GB

**Network:**
- **Bridge**: `vmbr1` (your private LAN, as configured in [OPNsense Setup](./opnsense-setup.mdx))
- **IPv4**: Static (e.g., `192.168.1.50/24`)
- **Gateway**: Your OPNsense LAN IP (e.g., `192.168.1.1`)
- **IPv6**: If not required, leave all IPv6-related fields empty.

**DNS:**
- **DNS Domain**: This is your local network's domain name (e.g., `home.arpa`, `yourdomain.local`). It's used for local name resolution within your network. If you don't have a local domain name, you can leave this field empty.
- **DNS Server**: This is the IP address of the DNS server(s) the container will use. If you're using OPNsense as your DNS, enter its IP here (e.g., `192.168.1.1`).

### 1.2 Container Configuration for Docker

After creating the container, but **before starting it**, edit the container configuration to enable Docker support:

```bash
# On Proxmox host, edit the container config (replace <CTID> with your container ID)
nano /etc/pve/lxc/<CTID>.conf
```

Add these lines to enable Docker support:

```
lxc.apparmor.profile: unconfined
lxc.cgroup2.devices.allow: a
lxc.cap.drop:
lxc.mount.auto: proc:rw sys:rw
```

Save the file (`Ctrl + O`, `Enter`, `Ctrl + X`) and start the container.

## 2. System Preparation

Once the container is started, you can access it via the Proxmox web interface console or SSH. Log in as the `root` user with the password you set during the container creation.

### 2.1 Update System

Enter the container console (via Proxmox web interface or SSH) and update the system:

```bash
# Update package lists and upgrade existing packages
apt update && apt upgrade -y
```

```bash
# Install basic utilities
apt install -y curl sudo git ca-certificates gnupg
```

### 2.2 Install Docker and Docker Compose

Install Docker using the official installation script:

```bash
# Add Docker's official GPG key
install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg -o /etc/apt/keyrings/docker.asc
chmod a+r /etc/apt/keyrings/docker.asc

# Add Docker repository
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/debian \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  tee /etc/apt/sources.list.d/docker.list > /dev/null

# Install Docker
apt update
apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Verify installation
docker --version
docker compose version
```

Expected output should show Docker version 20.x or higher and Docker Compose version 2.x or higher. 

```
...
Docker version 28.4.0, build d8eb465
Docker Compose version v2.39.4
```

## 3. Pangolin Installation

### 3.1 Method 1: Quick Install (Recommended)

:::info Official Quick Install Documentation
This quick install method is largely based on the official Pangolin documentation available at [https://docs.digpangolin.com/self-host/quick-install](https://docs.digpangolin.com/self-host/quick-install).
:::

Pangolin provides a quick installation script that automates the setup process:

#### 3.1.1 Download & Run the installer

```bash
# Run the quick install script
curl -fsSL https://digpangolin.com/get-installer.sh | bash
sudo ./installer
```

#### 3.1.2 Configure basic settings

The installer will prompt you for several basic settings:

- **Do you want to install Pangolin as a cloud-managed (beta) node? (yes/no)**: *My recommendation: `no`*
- **Base Domain**: Enter your root domain without subdomains (e.g., `yourdomain.com`). *My example: `kylehub.dev` (you should use your own base domain)*
- **Dashboard Domain**: Press Enter to accept the default `pangolin.yourdomain.com` or enter a custom domain (e.g., `pangolin.kylehub.dev`). *My recommendation: Press Enter (accept default `pangolin.kylehub.dev` or your equivalent)*
- **Let’s Encrypt Email**: Provide an email for SSL certificates and admin login. *My example: `admin@kylehub.dev` (replace with your actual email address)*
- **Do you want to use Gerbil to allow tunneled connections (yes/no) (default: yes)**: *My recommendation: Press Enter (accept default `yes`)*
- **Is your server IPv6 capable? (yes/no) (default: yes)**: *My recommendation: Press Enter (accept default `yes`)*
- **Would you like to run Pangolin as Docker or Podman containers? (default: docker)**: *My recommendation: Press Enter (accept default `docker`)*

#### 3.1.3 Configure email

:::info
Email functionality is optional and can be added later.
:::

Choose whether to enable SMTP email functionality:
*   **Default**: No (recommended for initial setup)
*   If enabled: You’ll need SMTP server details (host, port, username, password)

*My recommendation: Select `No` for email functionality during initial setup.*

#### 3.1.4 Start installation

Confirm that you want to install and start the containers:
*   The installer will pull Docker images (pangolin, gerbil, traefik)
*   Containers will be started automatically
*   This process takes 2-3 minutes depending on your internet connection
*   You’ll see progress indicators as each container is pulled and started.

*My recommendation: Proceed with the installation by selecting `Yes`.*

#### 3.1.5 Install CrowdSec (optional)

The installer will ask if you want to install CrowdSec for additional security:
*   **Default**: No (recommended for initial setup)
*   If enabled: You’ll need to confirm you’re willing to manage CrowdSec configuration

:::warning
CrowdSec adds complexity and requires manual configuration for optimal security. Only enable if you’re comfortable managing it. 
:::
:::info
CrowdSec can be installed later if needed. The basic installation provides sufficient security for most use cases.
:::


*My recommendation: Skip CrowdSec for now by selecting `No`.*

After these prompts, the installer will proceed to pull the necessary Docker images and start the Pangolin services. You will then see an overview of the running containers.


### 3.2 Method 2: Manual Docker Compose Installation

For more control over the installation process, you can manually set up Pangolin:

```bash
# Create installation directory
mkdir -p /opt/pangolin
cd /opt/pangolin

# Download docker-compose.yml
curl -fsSL https://raw.githubusercontent.com/digpangolin/pangolin/main/docker-compose.yml -o docker-compose.yml

# Create data directories
mkdir -p data/db data/redis

# Review the docker-compose.yml file (optional)
cat docker-compose.yml

# Start Pangolin
docker compose up -d

# Check status
docker compose ps

# View logs (useful for troubleshooting)
docker compose logs -f
```

### 3.3 Create Systemd Service

To ensure Pangolin starts automatically on boot, create a systemd service:

```bash
cat <<EOF > /etc/systemd/system/pangolin.service
[Unit]
Description=Pangolin Infrastructure Management
Requires=docker.service
After=docker.service

[Service]
Type=oneshot
RemainAfterExit=yes
WorkingDirectory=/opt/pangolin
ExecStart=/usr/bin/docker compose up -d
ExecStop=/usr/bin/docker compose down
TimeoutStartSec=0

[Install]
WantedBy=multi-user.target
EOF

# Enable and start the service
systemctl enable pangolin.service
systemctl start pangolin.service

# Check service status
systemctl status pangolin.service
```

#### 3.1.5 Install CrowdSec (optional)

:::warning
CrowdSec adds complexity and requires manual configuration for optimal security. Only enable if you’re comfortable managing it. CrowdSec can be installed later if needed. The basic installation provides sufficient security for most use cases.
:::

The installer will ask if you want to install CrowdSec for additional security:
*   **Default**: No (recommended for initial setup)
*   If enabled: You’ll need to confirm you’re willing to manage CrowdSec configuration

*My recommendation: Skip CrowdSec for now by selecting `No`.*

After these prompts, the installer will proceed to pull the necessary Docker images and start the Pangolin services. You will then see an overview of the running containers.

#### 3.1.6 Initial Access & Setup Token

Once the installation is complete, the installer will display a **Setup Token**. You must copy and save this token immediately, as it is required for the initial setup of Pangolin.

After obtaining the token, head over to the Pangolin web interface to complete the initial setup:

`https://pangolin.yourdomain.com/auth/initial-setup`

(Replace `pangolin.yourdomain.com` with your configured Dashboard Domain.)

## 4. Initial Configuration

### 4.1 Access Web Interface

Once Pangolin is running, access the web interface:

```
http://<pangolin-container-ip>:8080
```

For example, if your container IP is `192.168.1.50`:

```
http://192.168.1.50:8080
```

### 4.2 Complete Setup Wizard

1. **Create Admin Account**:
   - Set a strong username (avoid using `admin`)
   - Create a secure password (minimum 12 characters, include uppercase, lowercase, numbers, and symbols)
   - Provide an email address for notifications

2. **Configure Server Connection**:
   - Server Name: A descriptive name for your Pangolin instance
   - Server URL: The URL where Pangolin will be accessed (e.g., `http://192.168.1.50:8080` or `https://pangolin.yourdomain.com`)

3. **Database Setup**:
   - The quick install script handles this automatically
   - Verify database connectivity in the setup wizard

4. **Complete Initial Setup**:
   - Review settings
   - Click "Finish Setup"
   - You'll be redirected to the login page

### 4.3 First Login

Log in using the admin credentials you created during the setup wizard. You'll be greeted with the Pangolin dashboard.

## 5. Network Configuration

### 5.1 OPNsense Firewall Rules

To allow access to Pangolin from your LAN, configure OPNsense firewall rules:

1. Log in to OPNsense web interface
2. Navigate to **Firewall > Rules > LAN**
3. Click **Add** to create a new rule:
   - **Action**: Pass
   - **Interface**: LAN
   - **Protocol**: TCP
   - **Source**: LAN net (or specific IP ranges)
   - **Destination**: Single host or Network - `192.168.1.50` (Pangolin IP)
   - **Destination Port**: 8080
   - **Description**: Allow Pangolin Web Access from LAN
4. Click **Save**
5. Click **Apply Changes**

### 5.2 NGINX Proxy Manager Configuration (Optional)

For secure external HTTPS access to Pangolin, configure a reverse proxy using NGINX Proxy Manager:

:::info Reverse Proxy Benefits
Using a reverse proxy provides:
- SSL/TLS encryption (HTTPS)
- Domain-based access (e.g., `pangolin.yourdomain.com`)
- Centralized access control
- Protection from direct internet exposure
:::

#### 5.2.1 Add Proxy Host

1. Log in to NGINX Proxy Manager
2. Navigate to **Hosts > Proxy Hosts**
3. Click **Add Proxy Host**
4. Configure the following:

**Details Tab:**
- **Domain Names**: `pangolin.yourdomain.com`
- **Scheme**: `http`
- **Forward Hostname/IP**: Pangolin LAN IP (e.g., `192.168.1.50`)
- **Forward Port**: `8080`
- **Cache Assets**: No (optional, can enable later)
- **Block Common Exploits**: Yes (recommended)
- **Websockets Support**: Yes (important for real-time updates)

**SSL Tab:**
- **SSL Certificate**: Request a new SSL certificate (Let's Encrypt)
- **Force SSL**: Yes
- **HTTP/2 Support**: Yes
- **HSTS Enabled**: Yes (recommended)
- **HSTS Subdomains**: No (unless you need it)

5. Click **Save**

#### 5.2.2 DNS Configuration

Configure your DNS provider (e.g., Cloudflare) to point to your Pangolin instance:

**Option A: Direct DNS (A Record)**
- **Type**: A
- **Name**: `pangolin` (or your chosen subdomain)
- **Content**: Your public IP address
- **Proxy status**: Proxied (if using Cloudflare)

**Option B: Cloudflare Tunnel (Recommended for better security)**
- Set up a Cloudflare Tunnel pointing to your NGINX Proxy Manager
- Configure the tunnel to route `pangolin.yourdomain.com` to your Pangolin instance

#### 5.2.3 OPNsense WAN Rules (If Using Direct Access)

If you're exposing Pangolin directly via WAN (not recommended unless using Cloudflare Tunnel or VPN):

1. Navigate to **Firewall > Rules > WAN**
2. Click **Add**
3. Configure:
   - **Action**: Pass
   - **Interface**: WAN
   - **Protocol**: TCP
   - **Source**: Any (or restrict to specific IPs/countries)
   - **Destination**: WAN address
   - **Destination Port**: 443 (HTTPS)
   - **Description**: Allow HTTPS to NGINX Proxy Manager
4. Click **Save** and **Apply Changes**

:::caution Security Warning
Only expose services to the WAN if absolutely necessary. Always use strong authentication, SSL/TLS, and consider implementing additional security measures like fail2ban or IP whitelisting.
:::

## 6. Connecting Servers to Pangolin

### 6.1 Install Pangolin Agent

To monitor and manage servers with Pangolin, install the Pangolin agent on each target server:

1. In Pangolin dashboard, navigate to **Servers**
2. Click **Add Server**
3. Copy the provided installation command, which will look similar to:

```bash
curl -fsSL https://get.digpangolin.com/agent.sh | bash -s -- \
  --url=http://192.168.1.50:8080 \
  --token=<your-unique-token>
```

4. SSH into the target server and run the command
5. The agent will install and automatically connect to your Pangolin instance
6. Verify the connection in the Pangolin dashboard under **Servers**

### 6.2 Agent Configuration

The Pangolin agent configuration is typically located at `/etc/pangolin/agent.conf`. You can modify settings such as:

- Update frequency
- Monitored metrics
- Custom scripts
- Alert thresholds

After modifying the configuration, restart the agent:

```bash
systemctl restart pangolin-agent
```

## 7. Post-Installation Configuration

### 7.1 Configure Monitoring

Set up monitoring for your infrastructure:

1. Navigate to **Monitoring > Alerts**
2. Configure alert rules for:
   - CPU usage thresholds
   - Memory usage thresholds
   - Disk space warnings
   - Service availability
   - Docker container status

3. Set up notification channels:
   - Email notifications
   - Webhook integrations (e.g., Slack, Discord)
   - SMS alerts (if configured)

### 7.2 Deploy Containers via Pangolin

Pangolin can deploy and manage Docker containers across your infrastructure:

1. Navigate to **Containers > Deploy**
2. Choose deployment method:
   - **Docker Compose**: Paste your `docker-compose.yml` content
   - **Single Container**: Configure individual container settings
   - **Template**: Use pre-configured templates

3. Select target server(s)
4. Configure environment variables and volumes
5. Click **Deploy**

### 7.3 Backup Configuration

Implement a backup strategy for Pangolin data:

```bash
# Create backup script
cat <<'EOF' > /opt/pangolin/backup.sh
#!/bin/bash
BACKUP_DIR="/opt/pangolin/backups"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

# Create backup directory
mkdir -p "$BACKUP_DIR"

# Stop Pangolin
cd /opt/pangolin
docker compose down

# Backup data
tar -czf "$BACKUP_DIR/pangolin-backup-$TIMESTAMP.tar.gz" data/

# Restart Pangolin
docker compose up -d

# Remove backups older than 30 days
find "$BACKUP_DIR" -name "pangolin-backup-*.tar.gz" -mtime +30 -delete

echo "Backup completed: pangolin-backup-$TIMESTAMP.tar.gz"
EOF

# Make script executable
chmod +x /opt/pangolin/backup.sh

# Add to crontab (daily at 2 AM)
(crontab -l 2>/dev/null; echo "0 2 * * * /opt/pangolin/backup.sh >> /var/log/pangolin-backup.log 2>&1") | crontab -
```

### 7.4 Restore from Backup

To restore Pangolin from a backup:

```bash
# Stop Pangolin
cd /opt/pangolin
docker compose down

# Remove current data
rm -rf data/

# Extract backup
tar -xzf backups/pangolin-backup-YYYYMMDD_HHMMSS.tar.gz

# Restart Pangolin
docker compose up -d
```

## 9. Troubleshooting

### 9.1 Container Won't Start

If the LXC container fails to start or Docker services won't run:

**Verify Container Configuration:**
```bash
# Check container config on Proxmox host
cat /etc/pve/lxc/<CTID>.conf
```

Ensure these lines are present:
```
lxc.apparmor.profile: unconfined
lxc.cgroup2.devices.allow: a
lxc.cap.drop:
lxc.mount.auto: proc:rw sys:rw
```

**Check Container Options:**
- Verify that **Nesting** is enabled
- Verify that **Keyctl** is enabled
- Ensure the container is **Privileged**

### 9.2 Can't Access Web Interface

If you cannot access the Pangolin web interface:

**Check if Pangolin is running:**
```bash
docker compose ps
docker compose logs
```

**Verify network connectivity:**
```bash
# Check container IP
ip addr show

# Test internet connectivity
ping -c 4 8.8.8.8

# Test DNS resolution
nslookup google.com
```

**Verify firewall rules:**
- Check OPNsense LAN rules
- Verify no blocking rules in OPNsense
- Test access from different devices on your LAN

### 9.3 Docker Compose Errors

If you encounter Docker Compose version compatibility issues:

```bash
# Check Docker Compose version
docker compose version

# Update Docker Compose plugin
apt update
apt install --only-upgrade docker-compose-plugin

# Verify installation
docker compose version
```

### 9.4 Agent Connection Issues

If servers can't connect to Pangolin:

**Check Pangolin URL accessibility from target server:**
```bash
curl http://192.168.1.50:8080
```

**Verify firewall rules allow agent traffic:**
- OPNsense LAN rules
- Target server firewall (if any)

**Check agent logs on target server:**
```bash
journalctl -u pangolin-agent -f
```

**Regenerate agent token:**
1. In Pangolin dashboard, go to **Servers**
2. Remove the server
3. Add server again with a new token
4. Re-run installation command on target server

### 9.5 Database Connection Errors

If Pangolin reports database connection issues:

```bash
# Check database container
docker compose ps | grep db

# View database logs
docker compose logs db

# Restart database container
docker compose restart db

# If issues persist, recreate containers
docker compose down
docker compose up -d
```

## 10. Security Best Practices

### 10.1 Strong Authentication

- Use strong, unique passwords (minimum 16 characters)
- Enable two-factor authentication (if available in future versions)
- Regularly rotate passwords and API tokens
- Implement IP whitelisting where possible

### 10.2 Network Segmentation

- Keep Pangolin on your internal LAN (`vmbr1`)
- Use NGINX Proxy Manager for external access
- Implement Cloudflare Tunnel or VPN for remote access
- Never expose Pangolin directly to the internet without proper security measures

### 10.3 Regular Updates

Keep Pangolin and its dependencies up to date:

```bash
# Update Pangolin
cd /opt/pangolin
docker compose pull
docker compose up -d

# Update system packages
apt update && apt upgrade -y

# Update Docker
apt install --only-upgrade docker-ce docker-ce-cli containerd.io docker-compose-plugin
```

### 10.4 Log Monitoring

Regularly review Pangolin logs for suspicious activity:

```bash
# View Pangolin logs
docker compose logs --tail=100 -f

# Check for failed login attempts
docker compose logs | grep -i "failed\|error\|unauthorized"
```

## 11. Maintenance

### 11.1 Regular Health Checks

Create a health check script:

```bash
cat <<'EOF' > /opt/pangolin/health-check.sh
#!/bin/bash
echo "=== Pangolin Health Check ===$(date) ==="

# Check if Pangolin is running
if docker compose ps | grep -q "Up"; then
    echo "✓ Pangolin is running"
else
    echo "✗ Pangolin is not running"
    docker compose up -d
fi

# Check disk space
DISK_USAGE=$(df -h /opt/pangolin | awk 'NR==2 {print $5}' | sed 's/%//')
if [ "$DISK_USAGE" -gt 80 ]; then
    echo "✗ Warning: Disk usage is at ${DISK_USAGE}%"
else
    echo "✓ Disk usage is healthy (${DISK_USAGE}%)"
fi

# Check Docker service
if systemctl is-active --quiet docker; then
    echo "✓ Docker service is active"
else
    echo "✗ Docker service is not active"
fi

echo "==================================="
EOF

chmod +x /opt/pangolin/health-check.sh

# Schedule weekly health checks
(crontab -l 2>/dev/null; echo "0 8 * * 1 /opt/pangolin/health-check.sh >> /var/log/pangolin-health.log 2>&1") | crontab -
```

### 11.2 Log Rotation

Configure log rotation to prevent disk space issues:

```bash
cat <<'EOF' > /etc/logrotate.d/pangolin
/var/log/pangolin-*.log {
    daily
    rotate 14
    compress
    delaycompress
    missingok
    notifempty
}
EOF
```
